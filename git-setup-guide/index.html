    Git Installation & Setup Guide¶

    A step-by-step checklist to get you set up with Git and GitHub. Check off each step as you complete it.

    Target Audience

    This guide is designed for beginners who want to write code using either Windows 11 or macOS.

    Our Recommended Setup

    This guide is opinionated — we've made the choices for beginners so you can focus on learning.

    Choice Our Recommendation
    Terminal Use the one built into VS Code (or forks like Cursor, Windsurf)
    Shell Git Bash (Windows), Your default is good enough (macOS/Linux)
    Git commands Don't memorize them — use GUI clients like GitHub Desktop for daily work
    Authentication SSH keys (no passphrase for simplicity)
    Commit email Use your GitHub "noreply" email to keep your real email private
    Git hosting GitHub.com
    Why these choices?
    • Terminal - Use the one inside VSCode, or VSCode-like IDEs: Most beginners learn git because they are also learning code. Using the terminal built into VSCode or VSCode-like IDEs is a great way to get started.
    • Windows - Git Bash over PowerShell: The commands you learn (ls, rm, ssh-add, cat) work on macOS, Linux servers, and WSL — PowerShell cmdlets don't transfer.
    • GUI clients: Muscle memory for CLI commands comes later. GUIs reduce errors while you're learning concepts.
    • SSH with no passphrase: More secure than HTTPS. No passphrase simplifies setup for beginners. You can add a passphrase later for extra security.
    • Noreply email: Your email is visible in every commit's metadata. GitHub provides a private @users.noreply.github.com address to protect your inbox.

    Step 1: Open Your Terminal¶

    Open Your Terminal¶

    We recommend using VS Code or Antigravity as your development environment — both have built-in terminals.

    1. Install VS Code (if you haven't already)

    Download VS Code

    Windows Users: Which version? (x64 vs ARM)
    Windows Users: Which version? (x64 vs ARM)

    Most PCs use x64. Surface Pro X or Snapdragons might be ARM64.

    How to check:

    1. Press Win key, type "About your PC", and hit Enter.
    2. Look for System type.
    3. Download the matching installer.

    2. Open the Terminal

    1. Open VS Code
    2. Press Ctrl+` (Windows) or Cmd+` (macOS) to open the integrated terminal. An alternative is inside VS Code (top menu) > Terminal > New Terminal.
    Where is the Backtick (`) key?

    It is usually located above the Tab key and below Esc.

    Why VS Code?

    You'll be editing code anyway — having the terminal in the same window is convenient.

    How to close the terminal in VS Code?

    Sometimes, you need to restart the terminal. You can do this by pressing the trash can icon in the top right corner of the terminal.

    Step 2: Do You Have a GitHub Account?¶

    • Already have a GitHub account? → Skip to Step 3
    • Need an account? → Follow these steps:

    Create a GitHub Account¶

    1. Go to github.com/signup
    2. Enter your email, create a password, and choose a username
    3. Verify your email address
    4. (Optional) Set up 2FA for security: Settings → Password and authentication → Two-factor authentication

    Step 3: Install Git¶

    Choose your operating system:

    macOS Installation¶

    Check if Git is Already Installed¶

    Open your terminal and run:

    git --version
    • See a version number? (e.g., git version 2.39.0) → You're done! Skip to Step 4.
    • Get an error or prompt to install? → Continue below.

    Install Git¶

    The simplest way — no extra software needed:

    xcode-select --install

    A popup will appear — click Install and wait for completion (may take a few minutes).

    Verify Installation¶

    git --version

    You should see something like git version 2.39.0.

    Windows Installation¶

    Check if Git is Already Installed¶

    Open your terminal in VSCode and run:

    git --version
    • See a version number? (e.g., git version 2.42.0.windows.1) → You're done! Skip to Step 4.
    • Get an error? (e.g., 'git' is not recognized) → Continue below.

    Install Git¶

    1. Download from git-scm.com/download/win
    2. Run the installer
    3. Recommended settings during install:
      • ✅ Git Bash Here — adds right-click context menu
      • ✅ Use Git from Windows Terminal — so you can use git in PowerShell
      • ✅ Use bundled OpenSSH — for SSH key support

    Not sure which options to pick?

    The defaults are fine for most users. Just click "Next" through the installer.

    Verify Installation¶

    Open a new terminal window (important!) and run:

    git --version

    You should see something like git version 2.42.0.windows.1.

    Command not found after install?

    Close and reopen your terminal. The PATH is only updated for new terminal sessions.

    Step 4: GUI Client (Recommended)¶

    We recommend using a GUI (graphical) client — it's easier than the command line for beginners.

    GitHub Desktop — Free, beginner-friendly

    GitHub Desktop (Free)¶

    A visual Git client perfect for beginners — completely free.

    1. Download from desktop.github.com/download
    2. Install and open the app
    3. Click Sign in to GitHub.com
    4. Authorize GitHub Desktop in your browser

    Why GitHub Desktop?

    • No command line needed
    • See visual diffs before committing
    • Built by GitHub, so it integrates seamlessly
    Git Tower — Paid (Free for Students)

    Git Tower (Paid, Free for Students)¶

    A powerful Git client for professionals.

    • Paid: $69/year for individuals

    • Free for students: git-tower.com/students

    • Download from git-tower.com

    • Install and open the app
    • Go to Tower → Settings → Services
    • Click Add Account and sign in to GitHub

    Why Git Tower?

    • Advanced features like interactive rebase UI
    • Great for managing multiple repositories
    • Powerful conflict resolution tools

    Prefer the command line?

    That's fine too! The rest of this guide uses CLI commands, which work with or without a GUI client.

    Step 5: SSH Key Setup¶

    SSH keys let you securely connect to GitHub without entering your password each time.

    5a. Check for Existing Keys¶

    Check for Existing SSH Keys (macOS)¶

    Open your terminal and run:

    ls -la ~/.ssh

    Look for files named id_ed25519 or id_rsa (and their .pub counterparts).

    • Found keys? You can use your existing key — skip to 5c. Add SSH Key to GitHub
    • No keys or want a fresh one? Continue to "5b. Create SSH Key"

    Check for Existing SSH Keys (Windows)¶

    Open your terminal and run:

    In Git Bash:

    ls -la ~/.ssh

    Look for files named id_ed25519 or id_rsa (and their .pub counterparts).

    • Found keys? You can use your existing key — skip to "Add SSH Key to GitHub"
    • No keys or want a fresh one? Continue to "5b. Create a New SSH Key"

    5b. Create a New SSH Key¶

    Choose your operating system:

    Create SSH Key (macOS)¶

    1. Open Terminal and run:

      ssh-keygen -t ed25519 -C "your_email@example.com"

    2. Press Enter to accept the default file location

    3. Enter a passphrase (recommended) or press Enter for none

    4. Start the SSH agent:

      eval "$(ssh-agent -s)"

    5. Add your key to the agent:

      ssh-add ~/.ssh/id_ed25519

    6. Copy your public key:

      pbcopy < ~/.ssh/id_ed25519.pub

      Your public key is now in your clipboard!

    Create SSH Key (Windows)¶

    1. Open Git Bash and run:

      ssh-keygen -t ed25519 -C "your_email@example.com"

    2. Press Enter to accept the default file location

    3. Enter a passphrase (recommended) or press Enter for none

    4. Start the SSH agent:

      eval "$(ssh-agent -s)"

    5. Add your key to the agent:

      ssh-add ~/.ssh/id_ed25519

    6. Copy your public key:

      Get-Content ~\.ssh\id_ed25519.pub | Set-Clipboard

      Your public key is now in your clipboard!

    5c. Add SSH Key to GitHub¶

    1. Go to github.com/settings/keys
    2. Click New SSH key
    3. Give it a Title (e.g., "My MacBook" or "Work Laptop")
    4. Paste your public key into the Key field
    5. Click Add SSH key

    Test the connection:

    ssh -T git@github.com

    You should see:

    Hi username! You've successfully authenticated...

    First connection

    On first connection, you'll be asked to verify GitHub's fingerprint. Type yes to continue.

    Step 6: Configure Git Identity¶

    Configure Git Identity¶

    Tell Git who you are (use the same email as your GitHub account):

    git config --global user.name "Your Name"
git config --global user.email "your_email@example.com"

    Verify your settings:

    git config --global --list

    Per-project config

    Omit --global to set identity for a specific repository only.

    Step 7: Your First Repository¶

    Clone a Repository¶

    git clone git@github.com:username/repository.git

    Finding the SSH URL

    On GitHub, click the green Code button → select SSH → copy the URL.

    Make Your First Commit¶

    cd repository
# Make changes to files...
git add .
git commit -m "Your commit message"
git push

    Clone a Repository¶

    1. Open GitHub Desktop
    2. File → Clone Repository (or Cmd+Shift+O)
    3. Select the GitHub.com tab
    4. Choose a repository from your list (or paste a URL)
    5. Click Clone

    Make Your First Commit¶

    1. Make changes to files in your editor
    2. In GitHub Desktop, you'll see changed files in the left panel
    3. Add a Summary (commit message) at the bottom
    4. Click Commit to main
    5. Click Push origin to upload

    Clone a Repository¶

    1. Open Git Tower
    2. File → Clone (or Cmd+Shift+C)
    3. Paste the repository URL
    4. Choose where to save it
    5. Click Clone

    Make Your First Commit¶

    1. Make changes to files in your editor
    2. In Tower, go to the Working Copy view
    3. Stage files by clicking the + button
    4. Enter a commit message and click Commit
    5. Click Push to upload

    Quick Reference¶

    Common Git Commands¶

    Command What it does
    git status See what files have changed
    git add . Stage all changes
    git commit -m "message" Save staged changes
    git push Upload commits to GitHub
    git pull Download latest from GitHub
    git log --oneline View commit history
    git branch List branches
    git checkout -b name Create and switch to new branch

    Checklist Summary¶

    • GitHub account created
    • Git installed
    • GUI client installed
    • SSH key created and added to GitHub
    • Git identity configured
    • Successfully cloned a repository
    • Made your first commit

    Congratulations! You're ready to use Git and GitHub.
